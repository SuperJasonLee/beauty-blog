# 内容写作指南

## 如何使用 Markdown 编写文章

本博客使用 Markdown 格式编写文章，存储在 `content/posts/` 目录下。

## Front Matter 规范

每篇文章的文件开头必须包含以下 front matter：

```yaml
---
title: "文章标题"           # 必填，文章的完整标题
date: 2026-01-15             # 必填，发布日期
description: "文章摘要"      # 必填，将用于 meta description 和文章列表
categories: ["分类名称"]     # 必填，至少一个分类
tags: ["标签1", "标签2"]     # 推荐，有助于 SEO
draft: false                 # 草稿模式，true 时不会发布
featuredImage: ""            # 可选，特色图片路径
author: "整形美容知识百科"    # 可选，作者名称
---
```

## 写作规范

### 分级标题

- 文章标题使用 `##`（H2）
- 次级标题使用 `###`（H3）
- 所有页面有且仅有一个 H1（由 Hugo 自动生成）

### 内置短代码

**警告框**：
```
{{</* alert "warning" */>}}
警示内容...
{{</* /alert */>}}
```

**提示框**：
```
{{</* alert "info" */>}}
信息内容...
{{</* /alert */>}}
```

**小贴士**：
```
{{</* alert "tip" */>}}
建议内容...
{{</* /alert */>}}
```

**术语解释**：
```
{{</* term "术语名称" */>}}术语解释内容{{</* /term */>}}
```

### 图片

- 图片放在 `static/images/` 目录
- 使用 Markdown 图片语法：`![图片描述](/images/图片名.jpg)`

### 参考资料

文章末尾应列出参考资料：

```markdown
## 参考资料

1. 来源1
2. 来源2
```

### 免责声明

免责声明由系统在文章末尾自动追加，无需手动添加。
