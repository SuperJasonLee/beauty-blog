## Context

当前站点基于 Hugo + PaperMod 搭建，已部署上线。需要集成 Google AdSense 广告代码实现流量变现。站点模板使用 PaperMod 主题，部分已通过 layouts/ 目录覆盖。

## Goals / Non-Goals

**Goals:**
- 在全局 `<head>` 中注入 AdSense 自动广告脚本
- 在文章正文中插入内嵌广告位
- 在文章列表页中插入列表广告位
- 广告位在移动端自适应
- 通过 Hugo 配置管理 AdSense 发布商 ID，不硬编码

**Non-Goals:**
- 不涉及广告点击率优化（属于运营范畴）
- 不涉及多广告网络对比或 A/B 测试
- 不涉及广告拦截检测
- 不改变现有页面布局和内容结构

## Decisions

| 决策 | 选择 | 方案 | 理由 |
|---|---|---|---|
| 广告注入方式 | Hugo partial + config | 直接在模板中硬编码 | partial 方式可通过 hugo.yaml 的 params 配置发布商 ID，模板中引用，便于管理和切换 |
| 文章内嵌广告位置 | 文章正文第 3 段后 | 文章开头、结尾 | 第 3 段后既不影响开篇阅读体验，也能在用户阅读过程中展示，AdSense 政策合规 |
| 列表广告频率 | 每 3 篇文章后插入一个广告 | 每 5 篇 | 平衡广告密度与用户体验 |
| 广告类型 | 自动广告 + 手动广告位 | 仅自动广告 | 自动广告由 Google 算法优化展示位置，配合手动广告位可最大化收入 |
| 生产环境控制 | 仅生产环境加载广告脚本 | 所有环境都加载 | 避免开发/测试环境产生无效流量和违规计数 |

## Risks / Trade-offs

| 风险 | 缓解措施 |
|---|---|
| 广告影响页面加载速度 | AdSense 脚本使用异步加载，广告位使用懒加载 |
| 广告遮挡内容影响 AI 抓取 | 广告代码放在语义元素外部，使用 `<div>` 包裹，不使用 `<article>` 等语义标签 |
| AdSense 政策违规导致封号 | 严格遵守 AdSense 政策，不放置诱导点击广告，不伪装广告 |
| 广告导致页面布局偏移 (CLS) | 为广告位容器预留固定尺寸（由 AdSense 响应式广告自动处理），或使用 CSS min-height |
